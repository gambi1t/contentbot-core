## РЕШЕНО: база конструктора = этот форк (бывш. «maksim-bot»)

Было два разошедшихся форка: личный бот владельца (`content-bot`) и форк под
клиента (`maksim-bot`). Провели функциональный аудит обоих (4 параллельных
прохода по коду). Итог однозначный:

- **maksim-bot — надмножество content-bot, а не урезанная версия.** content-bot
  не содержит НИ ОДНОЙ продуктовой фичи, которой нет в коде maksim-bot.
- Уникально в maksim-bot: Remotion/auto-broll, пакет `carousel/`, пакет `broll/`,
  генератор идей (`idea_generator.py`), переработанные selfie/voice/монтаж,
  бренд-движок `BRANDS` + ~40 точек бренд-ветвления, более глубокие промпты.
- content-bot ≈ «бренд `default`» внутри maksim-bot (частный случай).
- То, что КАЗАЛОСЬ «есть только в content-bot», на деле: (а) **скрыто
  бренд-гейтом** (кнопки fal.ai «Фото»/«Видео», TikTok), (б) **не настроено**
  (OAuth-токены, ключи стоков), (в) **выключено** (4 команды закомментированы).
  Код везде на месте.

Поэтому **базой берём этот форк**. Брать content-bot = заново строить
бренд-слой и терять перечисленное выше. Идентичность репо отвязывается от
Максима: он становится **тенантом №1**, владелец (panferov) — **бренд `default`**.

> Оговорка: аудит шёл против локальной копии content-bot от 6 мая. Если боевой
> content-bot-сервер ушёл вперёд — снять дельту с него и портировать (это пункт
> в «первой работе», см. ниже).

## Первая работа (де-форк, не переписывание)

Чтобы этот форк стал чистой мультитенантной базой:
1. Раскомментировать 4 команды (`/update /report /launches /brand`) и сделать
   показ кнопок/команд **условным от бренда**, а не хардкод-комментарием.
2. Починить латентные «следы форка» (см. ниже «Известные дефекты»).
3. Снять дельту с боевого content-bot-сервера и портировать новое (если есть).
4. Вынести то, что варьируется на клиента, в конфиг тенанта (см. `03`).

## Известные дефекты (латентные, чинить при включении фич)

- **Гайды/лид-магнит:** публичный host вынесен в env `NOTION_GUIDES_PUBLIC_HOST`
  (было захардкожено на воркспейс владельца — кросс-тенант утечка). Сам
  системный промпт гайда всё ещё заточен под AI/Tesla-тематику — при включении
  гайдов у не-AI клиента промпт надо сделать бренд-зависимым. Фича сейчас ВЫКЛ
  у Максима (`NOTION_GUIDES_DB_ID` пуст).
- **Биллинг/баланс:** собран до MVP, но ВЫКЛ (`BILLING_ENABLED=0`), не обкатан,
  без платёжного шлюза (пополнение ручное). Это полноценная тенант-фича —
  см. `03`, раздел 8.

## Стек

- **Python**, Telegram-бот на `python-telegram-bot` (long polling).
- **ИИ**: Anthropic Claude (Opus для качества — сценарии/обложки/монтажный план,
  Sonnet — классификация/служебное). ~десяток вызовов на ролик.
- **Видео-аватар**: HeyGen (фото-аватар, Avatar IV / III).
- **Голос**: ElevenLabs (клон голоса, модели v2/v3).
- **B-roll графика**: Remotion (React/TSX → headless Chromium → MP4), отдельный
  проект, рендерит верхнюю половину split 1080×960.
- **Контент-план / CRM карточек**: Notion (per-tenant база).
- **Сборка видео**: ffmpeg (vstack split, Ken Burns, субтитры).
- **Кросспост**: YouTube Shorts / VK / Instagram (OAuth), TikTok (ручной пакет),
  Telegram-канал.
- **Деплой**: systemd-сервис на Hetzner, отдельный `.env` на инстанс, nginx для
  отдачи медиа/обложек по поддомену.

это доказательство, что вынос **возможен инкрементально** и есть с чего брать
пример стиля.

## Пайплайны (что бот реально делает)

1. **Идея → ролик**: идея/сценарий → структурирование в карточку Notion
   (классификатор рубрики) → обложка → озвучка (ElevenLabs) → видео-аватар
   (HeyGen) → B-roll → монтаж (ffmpeg) → кросспост.
2. **TG-пост**: идея → пост для Telegram-канала (отдельный writer, 5 типов постов).
3. **Карусель**: тема → 5 слайдов JSON → Playwright рендер PNG → в чат.
4. **Селфи**: пользователь записал говорящую голову → субтитры/обложка → кросспост.
5. **Банк идей / Launch monitor**: поиск виральных идей, дайджест.

## Где сейчас живёт мультитенантность (и почему это больно)

- **`BRANDS` — словарь в `bot.py`** (сейчас 3 ключа: `default`=владелец,
  `shoes`=другой клиент, `maksim`). Резолв через `_resolve_brand_cfg()` +
  ContextVar активного бренда. Активный бренд задаётся `DEFAULT_BRAND` в `.env`.
- **`.env` на инстанс** — токены, ключи, ID каналов, ID Notion-базы.
- **Внешние файлы промптов** — `script_prompt_maksim.txt` и т.п.
- **Зашитые литералы по всему `bot.py`** — и вот это корень боли. Несмотря на
  `BRANDS`, масса вещей была захардкожена под исходный аккаунт: рубрики, имена
  площадок, дескрипшены, few-shot примеры в промптах, классификатор. При форке
  под Максима это пришлось вычищать вручную по всему монолиту (см. лог де-форка).
  **Это и есть симптом отсутствия единой модели тенанта.** Что именно варьируется —
  в `03_TENANT_MODEL.md`.

## Деплой и среды

- Каждый клиент = отдельный systemd-сервис + свой `.env` + (сейчас) свой форк/сервер.
- Нет dev/staging-контура для безопасной разработки — это надо учесть в RFC
  (тебе дадут изолированный dev-инстанс, не прод).

## Что НЕ надо переписывать без причины

- `video_assembler.py` (ffmpeg-сборка) — работает, проверено на роликах.
- Remotion-связка (B-roll рендер 1080×960 + vstack) — рабочая архитектура.
- `tg_post_writer.py` — уже бренд-зависимый, можно брать как образец того, как
  должен выглядеть «вынесенный по тенанту» модуль.
