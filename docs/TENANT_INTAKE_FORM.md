# Анкета онбординга клиента на контент-бот

> Заполняется студией ВМЕСТЕ с клиентом ДО установки. Каждое поле → конкретная строка в `tenant.json` / `.env` / запись `BRANDS`, либо ассет/доступ. **[обяз.]** = без него бот не стартует/ломается ядро. **[опц.]** = нужно только под выбранную фичу. Нет точного числа → «уточнить», НЕ выдумываем (правило №1).
>
> Режим справа: 🟢 джун применит по чек-листу · 🔴 нужен разработчик · ✋ ручной внешний шаг.
>
> ⚠️ Главное про бренд: у НОВОГО клиента нет записи в `BRANDS`-dict, поэтому идентичность сначала заводит РАЗРАБОТЧИК (🔴), и только потом джун донастраивает. «Ноль кода» сегодня недостижим — см. «Оценка зрелости».
