# AI-видео (Kling): длительность — генерировать озвучку ПЕРВОЙ (audio-first)

> Спека для обеих сессий (Нолан/Илон) на одном ядре. Касается общих файлов
> `ai_video_broll.py` + `broll/handlers.py` → **координировать, кто реализует**
> (ветка Илона уже делает выбор 5/10с — этот подход дополняет его).
> Источник: прод-баг 25.06, бот Нолана. Версия 1 — 2026-06-25.

## Проблема (из прод-логов 25.06)
Сгенерировано 2 клипа Kling ×10с (=20с видео), а озвучка вышла **14.6с**:
```
[broll.assembler] очередь: 2 сегментов (по порядку, нужно 14.6s)
[broll.assembler] готово: montage.mp4, 14.6s
[subs] Audio for transcription: voiceover.mp3, duration=14.602449s
```
Ассемблер подрезал видео до 14.6с → **~6с оплаченного Kling-видео впустую** + кадры
рассинхронены с речью (визуал не туда по таймингу).

## Корень
Число клипов оценивается из **числа слов сценария** (`ai_video_broll.estimate_voiceover_sec`,
~150 слов/мин) ДО генерации озвучки. Порядок сейчас:
```
fullscreen_plan(script) → estimate_voiceover_sec(word_count) → clips_needed(estimate)
  → generate_ai_broll(N клипов)            # ПЛАТНО, по оценке
  → voiceover_fn(script)                    # реальная озвучка появляется ПОЗЖЕ
  → assemble(actual_voiceover_dur)          # подрезает под реальные 14.6с — поздно
```
Оценка из слов не совпадает с реальной озвучкой (темп речи, паузы, бренд-правки) →
перегенерация/обрез.

## Решение (согласовано Артёмом): AUDIO-FIRST
Сначала озвучка → замер реальной длины → план клипов ровно под неё:
```
voiceover_fn(script) → dur = _probe_duration(voiceover.mp3)   # реальные 14.6с
  → clips_needed(dur, clip_len)            # план ОТ факта, не от слов
  → generate_ai_broll(target по dur)       # генерим ровно сколько нужно
  → assemble                               # обрез минимальный/нулевой
```
Точность: подобрать число клипов + при необходимости **смешать длины 5/10с**, чтобы
сумма ≈ длине озвучки с минимальным остатком (например 14.6с → 10+5 вместо 10+10).

## Код (file:line — на 25.06)
- `ai_video_broll.py`: `estimate_voiceover_sec` (~282), `fullscreen_plan` (~313),
  `clips_needed`, `generate_ai_broll` (~346) — план от слов.
- `broll/handlers.py`: план (~600), генерация AI-видео (~643), озвучка
  `voiceover_fn` (~2004), сборка `assemble_broll_montage` (~2025).
- `broll/assembler.py`: `_probe_duration(voiceover_path)` (~216) — где меряется факт.

## Что сделать
1. В AI-видео-ветке flow: вынести генерацию озвучки ПЕРЕД планом клипов (или
   добавить вариант `generate_ai_broll_from_duration(actual_dur, ...)`).
2. `clips_needed`/выбор длины (5/10с) считать от `actual_dur`, а не от `estimate`.
3. Оценочный экран («сгенерирую N клипов ~$X») показывать уже от реальной длины
   озвучки (озвучка дешёвая/бесплатная относительно Kling — её не жалко сделать первой).

## Критерий приёмки
Длина итогового видео ≈ длине озвучки (обрез ≤ 1 клипа), нет оплаченных-и-выброшенных
клипов. Проверить на сценарии, где оценка слов и реальная озвучка расходятся (как 25.06).
