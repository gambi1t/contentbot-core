## #2 — Музыка: старые ТЕКСТОВЫЕ превью вместо 3 аудио-треков

**Симптом (запись SMM):** в пайплайне аватара/карточки выбор музыки показал старые
кнопки-«Трек 1 (45с)» БЕЗ аудио — нельзя прослушать.

**Корень (диагностика 3-агентным аудитом):**
- 3-трековый АУДИО-пикер (3× `send_audio` + кнопки выбора) есть ТОЛЬКО в:
  - Селфи: `selfie/handlers.py` → `_send_track_previews` (~845) + `selfie.music.pick_n_tracks(cat, n=3)` + `send_audio`.
  - B-roll: `broll/handlers.py` → `_send_broll_music_previews` (~741) + `_pick_n_tracks` + `send_audio`.
- Пайплайн аватара/карточки использует СТАРЫЙ ТЕКСТОВЫЙ пикер: `bot.py` хендлеры
  `music_pick:` / `music_cat:` / `music_apply:` (на maksim-prod ~19836-19945; у тебя
  ~14969/15317/20066) — только текст-кнопки, без `send_audio`.
- Доп: на финальном экране аватара кнопки музыки может не быть вовсе — юзер попадал в
  текстовый пикер карточки.

**Фикс (Q3 reuse + «все точки»):**
1. Переключить хендлеры `music_pick:`/`music_cat:` на 3-трековый аудио-пикер —
   переиспользовать `selfie.music.pick_n_tracks` + паттерн `_send_track_previews`
   (`send_audio` каждого трека + кнопка `…:pick:<cat>:<id>` под ним), как в селфи/B-roll.
2. **Все точки:** где предлагается музыка (аватар-финал, карточка) — должен быть ОДИН
   аудио-пикер. Если на финале аватара кнопки музыки нет — добавить (чтобы не было
   «пустого угла» / захода в старый текстовый).
3. Старый текстовый путь удалить/перенаправить на аудио-пикер.

**Критерий:** в любом пайплайне выбор музыки = 3 аудио-превью на прослушку (как в селфи).

## #4 — B-roll не сохраняется в карточку Notion

**Симптом (запись SMM):** B-roll (в т.ч. Kling 3.0) не попадает ссылкой в карточку Notion.

**Корень:** B-roll вообще не прикрепляет финал к карточке (не специфично для Kling).

  (~205/2121) кладёт финал в папку проекта (для кросспоста), но **блок-ссылку в карточку
  НЕ добавляет**.
- Селфи/аватар — добавляют: `bot.py` ~14706-14730: `save_media_permanent(final, "final_video")`

**Фикс (Q3 reuse):**
1. В `assemble_broll_from_draft` после доставки/моста — прикрепить ссылку на финал к
   карточке тем же паттерном, что в селфи (`save_media_permanent` + `notion.blocks.children.append`).
2. **DI:** `broll/handlers.py` НЕ импортирует notion-клиент/`save_media_permanent` (и не

3. **Все источники:** фикс в общей точке сборки → покрывает AUTO/HF/UPLOAD/MANUAL/AI_VIDEO

**Критерий:** после сборки любого B-roll в карточке Notion есть кликабельная ссылка на финал.
