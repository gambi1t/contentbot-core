## 1. Контекст продукта (1 минута)

Бот делает короткие вертикальные ролики (Reels/Shorts/VK) для предпринимателя Максима. Основные пайплайны: селфи+монтаж, AI-аватар (HeyGen)+монтаж, карусель, TG-пост. Пользователь — нетехнический (сам Максим и его SMM).

**Pipeline 2 «B-roll без аватара»** — ролик без лица: закадровый сценарий → озвучка клонированным голосом (ElevenLabs) → видеоряд из библиотеки живых клипов → субтитры. Работает в проде, двухфазный:

- **Фаза 1 (preview, дёшево):** тема → Claude пишет закадровый сценарий → селектор подбирает клипы → текстовый превью с кнопками «Собрать / Другой сценарий / Отмена». Черновик в `context.user_data["broll_draft"]` (in-memory, теряется при рестарте — осознанно).
- **Фаза 2 (assemble, после approve):** озвучка → ffmpeg-монтаж → субтитры → MP4 в чат.

Инфраструктура: python-telegram-bot v21, ОДИН процесс (рендеры в `asyncio.to_thread`), сервер 4 vCPU / 7.6 GiB. LLM — Claude через подписочный CLI-врапер (текст-в-текст, без инструментов). Время сборки до 20 минут приемлемо (подтверждено владельцем).

## 2. Что уже есть и переиспользуется (НЕ переписываем)

### 2a. Селектор клипов (metadata-aware, Claude-видеоредактор)
У каждого клипа библиотеки JSON-сайдкар: категория (karting/glamping/sup/personal), текстовое описание кадра, теги, quality_grade (broll=универсальный, event=событийный, weak=отсев). Claude получает сценарий + текстовый каталог, возвращает упорядоченные индексы:

```
[{i}] сцена={scene} grade={grade} | {description} | теги: {tags}

ПРАВИЛА (из промпта):
- Выбери от {n_min} до {n_max} клипов (дефолт 5-9).
- Порядок в ответе = порядок появления в ролике. Первый — цепляющий, под хук.
- Клипы должны поддерживать смысл сценария (бизнес→рабочие кадры, драйв→трасса...).
- grade=broll предпочтителен; event — только если прямо по теме.
- Не повторяться, разнообразить планы.
ФОРМАТ: строго JSON-массив индексов: [3, 17, 0, 22, 9, 14]
```
Есть фоллбэк без LLM (keyword-матчинг по сценам) — ролик соберётся даже при сбое Claude.

### 2b. Ассемблер (всеядный по входу)
`assemble_broll_montage(clip_paths, voice_path, out, work_dir)`: меряет длительность озвучки → нормализует КАЖДЫЙ клип в 1080×1920 (blurred-pad: фон — размытая копия, ничего не кропается, подходит любым aspect) → сегменты ~равной длины под озвучку, очередь зацикливается при нехватке → concat → mux озвучки. Ключевое: **ассемблеру всё равно, откуда клип** — любые mp4 в `clip_paths` будут смонтированы.

```python
def _normalize_clip(src, seg_len, dst):
    # blurred-pad 1080×1920, fps=30, -an, ultrafast (промежуточный сегмент)
    "[0:v]split=2[bg][fg];[bg]scale=270:480:...,boxblur=8:1,scale=1080:1920[bgb];"
    "[fg]scale=1080:1920:force_original_aspect_ratio=decrease[fgs];"
    "[bgb][fgs]overlay=центр,setsar=1,fps=30[outv]"
```

### 2c. Генерация графики HyperFrames (single-shot, недавно переведена с агентного билда)
`hyperframes_broll.py`: storyboard-фаза (Claude планирует N разных сцен, machine-валидатор разнообразия: архетипы hero_number/before_after_cards/checklist/..., запрет повторов) → **single-shot генерация HTML-сцен** (одна completion на сцену, Opus 4.8, ~46-75 с/сцена, параллель semaphore=2) → layout-детектор (puppeteer: offscreen/overlap/crowding) → fix-rounds (перегенерация только проблемных сцен) → рендер своим puppeteer+ffmpeg (5 с, 1080×1920, 30fps). Прогресс-уведомления по фазам уже встроены (callback из рабочего потока через `run_coroutine_threadsafe`).

⚠️ Нюанс: текущий планировщик сцен (`build_bookend_montage_plan` и storyboard-промпт) заточен под формат «хук-АВАТАР → вставки → CTA-АВАТАР» и под тайминг аватарного видео. Для безаватарного ролика опоры-аватара нет.

### 2d. Готовые UI-компоненты (из соседних пайплайнов)
- **Рич-пикер библиотеки**: превью клипов/фото медиагруппой + мультивыбор тогглами + «Готово (N)» + «Моё выбранное» (обзор/удаление набора). Используется в селфи и карточке.
- **Upload-флоу**: приём фото/видео от юзера с прогрессом, лёгкие превью (фикс 413), сохранение в проект + опционально в библиотеку с категорией.
- Озвучка, субтитры (с бизнес-словарём против ослышек Whisper), кросс-постинг — общие.

## 3. План фичи (что хотим построить)

Сейчас Фаза 1 жёстко зовёт авто-селектор. Хотим после генерации сценария дать выбор **источника видеоряда** (требование владельца, дословно: «загрузить свои фото/видео / выбрать из библиотеки вручную / Клод сам решает» + графика):

```
Тема → сценарий → ЭКРАН «Видеоряд?»:
  1. 🤖 Авто из библиотеки   — текущее поведение (селектор 2a)
  2. 👆 Выбрать вручную      — рич-пикер (2d)
  3. 📤 Загрузить свои       — upload-флоу (2d), фото+видео
  4. 🎨 Только графика HF    — storyboard → N сцен → рендер → clip_paths
  5. 🔀 Авто + графика       — живые из селектора + 2-3 HF-вставки вперемешку
→ превью (сценарий + сводка видеоряда) → «Собрать» → Фаза 2 (без изменений)
```

Принятые нами решения (можно оспаривать):
- HF-сцены попадают в ролик просто как элементы `clip_paths` (ассемблер всеядный) — без изменения монтажа.
- Для микса: позиции HF-вставок в очереди решает Claude-селектор? Или вперемешку детерминированно (например, каждая 3-я)? — пока НЕ решено, см. вопрос Q4.
- Для «только HF»: число сцен из длительности озвучки (~5 с/сцена → 30-сек озвучка ≈ 6 сцен).
- Storyboard для HF строим из ЗАКАДРОВОГО сценария (не из аватарного тайминга): смысловые биты текста → сцены.
- Фото при ручном выборе/загрузке оживляются Ken Burns (механизм есть в соседнем ассемблере — но в Pipeline 2-ассемблере фото пока НЕ поддержаны, только видео. Нужно решить: поддержать или ограничить «только видео»? см. Q6).
- Лейбл кнопки «(в разработке)» убираем.

## 4. Известные нам ограничения и риски

1. **Один процесс**: тяжёлые рендеры в `to_thread`, кнопки бота могут подлагивать во время ffmpeg. Воркер-процесс — осознанно отложен (P2).
2. **Черновик в user_data**: при рестарте бота между «выбрал источник» и «Собрать» — черновик теряется (юзер начинает заново). Для нынешнего одношагового флоу это было ок; с развилкой шагов становится больше — может стать болезненнее.
3. **HF-время**: только-HF ≈ storyboard (1-2 мин) + 6 сцен (~3-5 мин) + рендер (~4-5 мин) + fix-rounds (0-12 мин) = 8-25 мин. Прогресс-уведомления есть. Владелец ок до 20 мин.
4. **Claude-подписка шарится** с остальными вызовами бота (rate-limit five_hour / seven_day_opus). Pre-flight зонд лимита есть в HF-модуле.
5. **Превью видеоряда текстовое** (категории + счётчик), без визуальных миниатюр выбранного. Для «вручную» юзер видел превью в пикере; для «авто/HF» — не видит кадры до готового ролика.

## 5. Вопросы к ревьюеру

- **Q1 (меню/UX):** 5 кнопок источника после сценария — норм для нетехнического юзера, или свернуть (например, «Авто / Вручную / Ещё...»)? Есть ли лучший паттерн для развилки «живое/графика/микс» в подобных продуктах?
- **Q2 (состояние):** многошаговый draft в `user_data` (in-memory) — где это укусит при развилке (upload-флоу может длиться минуты, HF — десятки минут)? Стоит ли уже сейчас минимальный durable-draft (json на диске), или это преждевременно?
- **Q3 (HF-storyboard без аватара):** как правильнее нарезать ЗАКАДРОВЫЙ сценарий на сцены для графики: по предложениям/смысловым битам самим Claude'ом (ещё один LLM-вызов) или арифметически (длительность озвучки / 5 с)? Что даёт лучшую синхронизацию смысла и картинки?
- **Q4 (микс):** кто решает позиции HF-вставок среди живых клипов: (а) селектор-Claude видит и живые, и описания будущих HF-сцен в одном каталоге и упорядочивает всё вместе; (б) детерминированная решётка (каждый 3-й слот — графика); (в) сначала живые, HF в конце как акценты? Плюсы/минусы, что выбрали бы вы?
- **Q5 (число HF в миксе):** фикс 2-3 или пропорция от длительности? Когда графики «слишком много» для зрителя?
- **Q6 (фото):** поддержать фото в Pipeline 2 (через Ken Burns, код есть в соседнем ассемблере) сразу или v2? Аргументы за/против с точки зрения скоупа.
- **Q7 (надёжность):** какие фоллбэки обязательны: HF-генерация упала на середине микса → собрать только из живых? upload-флоу брошен на полпути → таймаут черновика? Что ещё?
- **Q8 (вы упустили):** что мы не увидели — гонки, edge-cases, UX-провалы, расширения, которые дёшево заложить сейчас?
