# Журнал прод-изменений (ветка `maksim-prod`)

> Этот файл — для человека, а не для парсера. Здесь простым языком: что
> поменялось в живом боте, **зачем**, **где это в коде** и **как проверено**.
> Голый `git diff` смотреть не обязательно — сначала прочитай сюда.

## Что это за ветка и как она относится к репозиторию

В репозитории `contentbot-core` две роли:

- **`main`** — твоя база для конструктора (мультитенантный движок). Мы её
  **не трогаем**. Ты строишь поверх неё.
- **`maksim-prod`** — снимок **живого прод-бота Максима** (`@yumsunov_content_bot`).
  Сюда мы выкладываем новые рабочие фичи по мере того, как доделываем их в
  проде. Это **референс**: «вот так фича реально работает в бою» — чтобы тебе
  было от чего отталкиваться, когда переносишь её в модульную архитектуру
  конструктора.

**Договорённость:** прод-фичи всегда едут в `maksim-prod`, **никогда в `main`**.
Переносом фич в конструктор управляешь ты — в своей структуре, когда дойдёшь.
Так наш прод и твоя база не конфликтуют.

## 2026-05-23 — «спайн» пайплайна, Срез 1a (фундамент, на моках)

Начало выноса пайплайна «идея → ролик» в **отдельный модуль-оркестратор**
`content_pipeline/` (параллельный трек, не вплавляли в `bot.py`). Архитектура
согласована двумя раундами внешнего ревью. Прод **не затронут** — это новый
изолированный пакет.

- **Что есть:** SQLite-store (runtime-правда; Notion — зеркало), модели
  `PipelineEvent` / `UIIntent` / `EffectCommand`, state-machine `PipelineSpine`,
  cost-gate с идемпотентностью, resume + stale-guard через CAS по `stage_version`.
  Линейный путь idea→script→cover→voice→**avatar cost-gate**. Шаги — мок (без
  реального HeyGen).
- **Доказано (14 unit-тестов зелёные):** после рестарта/`/start` run
  восстановим; платный avatar-job нельзя запустить случайно или дважды; ядро не
  импортирует `bot.py` и не содержит клиент-специфики.
- **Контракт для конструктора:** `docs/pipeline_spine/CONTRACT.md` (события на
  вход, UIIntent/EffectCommand на выход, схема store, 10 инвариантов).
- **Следующее (1b):** Telegram-адаптер (скрытая точка входа) + извлечение
  реальных `ScriptService`/`CoverService`/voice из `bot.py` + Telethon-смок до
  gate. Реальный HeyGen, планы selfie/broll, публикация — дальше.

## 2026-05-22 — блок «качество пайплайна + свой голос»

Все правки задеплоены на прод (сервер `nox-maksim`) и прогнаны через
Telethon-тесты (живой бот, реальные нажатия) — все зелёные.

### 1. Фича «свой голос» — аватар говорит голосом пользователя
**Зачем.** Раньше аватар озвучивался только синтезом (TTS / клон голоса).
Максим хотел иногда наговаривать сам, чтобы аватар липсинкался под его реальную
запись.
**Что сделано.** На шаге выбора версии аватара появилась кнопка
«🎤 Озвучить своим голосом». Пользователь шлёт голосовое → оно грузится в HeyGen
как аудио-дорожка (`voice: {type: "audio", audio_url}`) → аватар синхронизируется
под неё, без TTS.
**Где в коде.** `bot.py`: `_consume_selfvoice_audio`, `_render_avatar_from_audio`,
callback `heygen_selfvoice:`, ветка `awaiting_selfvoice` в `process_voice`.
**Тест.** TDD-смоук `_smoke_own_voice_avatar.py` — реальный рендер audio→avatar
прошёл (HeyGen вернул completed). Кнопки — Telethon `10_selfvoice_button`.
На шаге версии теперь 3 кнопки: **Avatar 3** (дефолт-для-тестов, дешевле) /
**Avatar IV** / **Озвучить своим голосом**.

### 2. Пикер обложки — выбор фона из трёх фото
**Зачем.** Раньше на шаге обложки бот показывал ОДНО случайное фото, и его
приходилось перещёлкивать по одному. Неудобно.
**Что сделано.** Теперь бот шлёт **альбом из 3 случайных фото** пула + кнопки
`[1] [2] [3]`, «🔄 Другие фото» (переролл) и «📤 Загрузить своё фото». Выбрал
номер → дальше как раньше: 5 текстов обложки.
**Где в коде.** `bot.py`: `_show_cover_photo_picker`, callback `cover_photo:N` и
`cover_photo:reload`, `_send_cover_text_options`. Пул фото —
`assets/avatars/<бренд>/`.
**Тест.** Telethon `11_cover_photo_picker` — полный путь: идея → сценарий →
утвердить → альбом + кнопки → переролл → выбор «1» → тексты обложки.

### 3. B-roll фото из библиотеки попадают в «Скачать материалы»
**Зачем.** Когда монтаж брал фоновые фото из общей библиотеки, в ZIP «Скачать
материалы» их не было — клиент не мог домонтировать ролик в CapCut с теми же
исходниками.
**Что сделано.** Фото библиотеки, **реально использованные** в монтаже,
копируются в папку проекта (`projects/<id>/photos/`, префикс `broll_`) → теперь
попадают в ZIP.
**Где в коде.** `video_assembler.py`: `_gather_photo_broll` (копирование
выбранных исходников перед сборкой Ken-Burns клипов).

### 4. Обложка ставится баннером Notion-карточки
**Зачем.** Карточки в доске `/cards` были без визуала. У карточек из «Банка
идей» обложка вообще не доезжала до Notion (карточка создаётся раньше, чем
генерится обложка).
**Что сделано.** Обложка теперь ставится как **page-level cover** (баннер
карточки), а не только картинкой в теле страницы. Для заранее созданных карточек
баннер дописывается на шаге сохранения обложки.
**Где в коде.** `bot.py`: `create_notion_card` (параметр `cover`) + ветка
`cover_confirm` для существующих страниц (`pages.update(cover=…)`).

### 5. Мелочи прод-качества
- **Себестоимость ролика** пишется в лог после сборки: аватар (сек), озвучка
  (символы), B-roll (клипы), финал (МБ) — `bot.py`, `card_asm_go`.
- **Защита от 413** (Telegram «файл слишком большой») на финальном монтаже +
  сжатие превью аватара.
- **Удалён мёртвый код** `generate_montage_plan` (LLM-планировщик монтажа,
  больше не вызывается — монтаж детерминированный).

### 6. Безопасность (важно прочитать)
В `scripts/add_brand_property.py` и `scripts/set_brand_on_old_cards.py` раньше
был **захардкоженный Notion-токен**. Перевели на `os.getenv("NOTION_TOKEN")`.
Скрипты одноразовые (миграции, уже отработали). Никаких секретов в коде быть не
должно — всё через переменные окружения (`.env`), который в git **не** коммитится
(см. `.gitignore`).

### 7. Пост-ревью фиксы (после независимой проверки субагентом)
- **Пикер обложки: защита от «протухших» кнопок.** Если бот перезапустили или
  пользователь нажал `/start` (сессия `pending` потерялась), нажатие на старую
  кнопку выбора фото молча падало (None-данные) — кнопка «умирала» без реакции.
  Теперь показывает понятный алерт «Подборка устарела — открой карточку заново».
  `bot.py`, ветка `cover_photo:` в обработчике колбэков.
- **Лейбл версии в «свой голос».** Статус показывал «Avatar 4» для Avatar IV —
  поправлено на корректную карту версий. `bot.py`, `_render_avatar_from_audio`.
