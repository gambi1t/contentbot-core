## Твоя задача в двух предложениях

Сейчас, чтобы запустить бота под нового клиента, уходит **~40 часов ручной адаптации**. Нужно спроектировать **конструктор**: новый клиент = заполнить конфиг + загрузить ассеты, **без правок кода**, на работающей кодовой базе, без второй параллельной версии продукта.

## Что мы ждём от тебя (и в каком порядке)

| Этап | Артефакт | Гейт |
|---|---|---|
| 1 | **RFC** (проектный документ, без кода) по шаблону `04` | Согласование с владельцем + AI-ревью |
| 2 | **PoC**: один модуль, вынесенный по твоей архитектуре, работает end-to-end | Демо + ревью |

Реализацию всего конструктора в этот заход **не ждём** — её строим дальше по твоему RFC. Твой вклад — спроектировать и доказать подход на одном модуле.

## Три жёстких правила (нарушение = переделка)

1. **Сначала RFC, потом код.** Большие решения принимаются на бумаге. Код без согласованного RFC не принимаем.
2. **Никаких параллельных версий.** Живой бот — это baseline. Работаем ветками от него, не плодим «вторую систему», которую потом надо чинить параллельно.
3. **Прод не трогаем.** Доступа к прод-серверу и боевым секретам у тебя нет и не будет (см. `05`, раздел «Доступ»). Это про данные клиентов и биллинговые ключи, не про доверие.
