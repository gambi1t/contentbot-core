**Главная ошибка новичков в глэмпинге — отсутствие концепции**

Когда говорят «новички в глэмпинге», нужно понимать, что есть две
разные категории.

Первая — это ребята, которые строят не глэмпинг, а гостевые дома.
Обычно это СНТ, ЛПХ, ИЖС. На участке появляется один-два, максимум
три домика — и это нормально. Но это не глэмпинг.

И главная ошибка, которая встречается у большинства новичков —
отсутствие концепции. Про это редко кто говорит, но именно здесь
и кроется большая ошибка.

У любого глэмпинга должна быть идея, ради которой люди туда едут.
Не просто «домики в лесу» — а конкретный сценарий: что человек
получит, чего нет в десяти других местах.

Без концепции вы конкурируете ценой. С концепцией — вас находят
сами.

Глэмпинг — это не бизнес пока у вас нет системы и команды.

## Архитектура интеграции в код

В `tg_post_writer.py` я бы сделал так:

```python
SYSTEM_PROMPT_ARTEM = """..."""    # текущий (без изменений)
SYSTEM_PROMPT_MAKSIM = """..."""   # новый, выше

def _get_system_prompt(brand: str) -> str:
    if brand == "maksim":
        return SYSTEM_PROMPT_MAKSIM
    return SYSTEM_PROMPT_ARTEM

# В generate_post():
def generate_post(inp, claude_client, *, max_tokens=1500):
    from bot import _get_active_brand_name
    system_prompt = _get_system_prompt(_get_active_brand_name())
    response = claude_client.messages.create(
        model="claude-opus-4-20250514",
        max_tokens=max_tokens,
        system=system_prompt,
        messages=[{"role": "user", "content": _build_user_message(inp)}],
    )
    ...
```

Артёмов код не трогается, его SYSTEM_PROMPT остаётся как есть.
