# contentbot-core

Кодовая база контент-бота — фабрика коротких видео (идея → сценарий → ИИ-аватар →
озвучка → B-roll → монтаж → кросспост). Это **база будущего мультитенантного
конструктора**: один движок, клиент = конфиг + ассеты, без правок кода.

## С чего начать (для разработчика)

📖 **Читай `docs/00_START_HERE.md`** — это точка входа. Дальше по порядку
`docs/01`…`docs/06`.

Кратко:
- `docs/01_BUSINESS_BRIEF.md` — зачем конструктор (запрос от бизнеса).
- `docs/02_ARCHITECTURE_CONTEXT.md` — как устроено сейчас + решение по базе.
- `docs/03_TENANT_MODEL.md` — что варьируется от клиента к клиенту.
- `docs/04_RFC_TEMPLATE.md` — формат RFC, который ждём первым (до кода).
- `docs/05_PROCESS_GUARDRAILS.md` — процесс, гейты, доступы.
- `docs/06_ARCHITECT_REVIEW_ADOPTED.md` — внешнее арх-ревью + скоуп.
- `docs/examples/tenant.example.yaml` — синтетический конфиг тенанта (референс).

## Важные правила

- **Сначала RFC, потом код.** Большие решения — на бумаге.
- **Никаких параллельных версий.** Эта база — единственный источник правды.
- **Прод не трогаем.** Работа только на dev-инстансе + синтетическом тенанте
  (без боевых секретов и данных реальных клиентов). См. `docs/05`.
- **Секреты — только в `.env`** (он в `.gitignore`, в репо его нет). Никаких
  токенов/ключей в коде.
